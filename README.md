# Norbert AI — платформа семантического консультирования с графом знаний

**Norbert AI** — это готовая архитектура для создания интеллектуальных ассистентов, которые умеют:

- загружать и обрабатывать большие коллекции документов (PDF, DOCX, TXT);
- извлекать из них структурированные факты и связи с помощью LLM;
- хранить их в графовой базе (Neo4j) и поисковом движке (Elasticsearch);
- отвечать на сложные запросы пользователей с указанием источников, выделением ключевых параметров и определением пробелов в знаниях.

В качестве демонстрационного примера система настроена на **горно-металлургическую** тематику, но архитектура спроектирована так, чтобы её можно было легко адаптировать для медицины, юриспруденции, финансов, нефтегаза или любой другой предметной области — достаточно заменить данные, схему графа и промпты для LLM.

---

## 🔥 Возможности

- **Семантическая обработка документов** – извлечение текста, разбивка на смысловые фрагменты.
- **Гибридный поиск** – сочетание внутреннего полнотекстового поиска (Elasticsearch) с внешним веб-поиском (DuckDuckGo + Crossref) для актуализации ответов.
- **Граф знаний** – построение графа в Neo4j: документы, авторы, параметры, условия, связи.
- **LLM-анализ** – на базе GitHub Models (DeepSeek‑R1) с резервом через Yandex GPT:
  - извлечение триплетов (субъект – предикат – объект);
  - выявление числовых параметров, операторов, единиц измерения, географии, годов;
  - формирование структурированных ответов с оценкой уверенности и ссылками на источники.
- **Интерактивный веб-интерфейс** – на Streamlit с историей запросов.
- **Готов к развёртыванию** – всё упаковано в Docker Compose.

## 🧩 Архитектура

Проект состоит из двух основных конвейеров (пайплайнов):

### 1. Пайплайн загрузки и индексации (ETL)

- `extract_text.py` – извлекает текст из файлов.
- `chunking.py` – разбивает текст на перекрывающиеся чанки.
- `nlp_pipeline.py` – асинхронно отправляет чанки в LLM для извлечения фактов.
- `build_graph.py` – загружает триплеты в Neo4j и генерирует визуализацию (pyvis).
- `index_elastic.py` – индексирует чанки и метаданные в Elasticsearch.

### 2. Пайплайн обработки запросов (онлайн)

- Бэкенд на FastAPI (`main.py`) координирует:
  - парсинг запроса (выделение условий, параметров, географии, лет);
  - гибридный поиск (Elasticsearch + веб);
  - обход графа (Neo4j) для получения связанных фактов;
  - синтез финального ответа через LLM.
- Фронтенд на Streamlit общается с API и отображает результат.

Компоненты слабо связаны, их можно заменять или расширять независимо.

## 🚀 Быстрый старт

### Требования
- Docker и Docker Compose
- Python 3.10+ (для локальной разработки)
- API-ключ GitHub Models (или Yandex GPT как запасной)

### Переменные окружения
Создайте в корне файл `.env`:

```env
# Обязательные
GITHUB_TOKEN=ваш_токен_github_models
ELASTICSEARCH_HOST=elasticsearch:9200
NEO4J_URI=bolt://neo4j:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=ваш_пароль

# Опционально (веб-поиск)
DUCKDUCKGO_ENABLED=true
CROSSREF_ENABLED=true
```
Запуск через Docker
bash

docker-compose up --build

После запуска:

    Интерфейс пользователя: http://localhost:8501

    API: http://localhost:8000

    Документация API: http://localhost:8000/docs

Загрузка собственных документов

Поместите файлы (PDF, DOCX, TXT) в папку data/raw/ и выполните:
bash

python extract_text.py
python chunking.py
python nlp_pipeline.py
python build_graph.py
python index_elastic.py

🛠️ Адаптация для другой предметной области

Чтобы перенастроить систему под новые задачи (например, медицинские публикации, юридические контракты, финансовые отчёты), достаточно изменить:

| Компонент | Где править | Пример изменений |
|-----------|-------------|------------------|
| **Источник данных** | Заменить файлы в `data/raw/` или модифицировать `extract_text.py` для загрузки из API. | Вместо PDF — XML из PubMed. |
| **Схема графа** | Изменить типы узлов и связей в `build_graph.py` и промпты извлечения триплетов в `nlp_pipeline.py`. | Добавить узлы "Пациент", "Лекарство". |
| **Промпты LLM** | Системный промпт и инструкции в `llm_client.py` и `nlp_pipeline.py`. | Попросить LLM извлекать дозировки и противопоказания. |
| **Парсинг запроса** | Функции в `utils.py` (обнаружение условий, параметров, географии). | Распознавать финансовые коэффициенты или юридические нормы. |
| **Поисковые фильтры** | Маппинг Elasticsearch и построение запросов в `index_elastic.py`. | Добавить поле "фаза клинического исследования". |
| **Тексты интерфейса** | Streamlit-виджеты и подписи в папке `ui/` (если отделена). | Переименовать поля для новой доменной области. |

Все настраиваемые параметры (размер чанка, перекрытие, лимиты токенов, пути) вынесены в config.py, который можно редактировать или переопределять через переменные окружения.

📊 Технологический стек

    Бэкенд: FastAPI, Uvicorn

    Фронтенд: Streamlit

    Базы данных: Neo4j (граф), Elasticsearch (поиск), SQLite (история)

    LLM: GitHub Models (DeepSeek‑R1) + Yandex GPT (резерв)

    Веб-поиск: DuckDuckGo, Crossref

    NLP: spaCy (опционально), LangChain Text Splitters

    Контейнеризация: Docker и Docker Compose

    Визуализация: pyvis
